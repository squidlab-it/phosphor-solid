# Contributing to @squidlab/phosphor-solid

Thanks for your interest in contributing! This guide will help you get started.

## Prerequisites

- [Node.js](https://nodejs.org/) (LTS recommended)
- [npm](https://www.npmjs.com/)

## Setup

```sh
git clone https://github.com/AlessioGresworWorked/phosphor-solid.git
cd phosphor-solid
npm install
```

## Project structure

```
scripts/generate.ts   # Code-generation script (reads @phosphor-icons/core SVGs)
src/icons/            # Generated icon components (do NOT edit by hand)
src/mod.tsx           # Barrel export
biome.json            # Formatter and linter config
```

> **Important:** files inside `src/icons/` are auto-generated by `scripts/generate.ts`.
> Do not modify them directly — your changes will be overwritten.

## Available scripts

| Command             | Description                          |
| ------------------- | ------------------------------------ |
| `npm run generate`  | Regenerate all icon components       |
| `npm run check`     | Type-check with `tsc --noEmit`       |
| `npm run fmt`       | Format with Biome                    |
| `npm run fmt:check` | Check formatting without writing     |
| `npm run lint`      | Lint with Biome                      |

## Making changes

1. **Fork** the repository and create a new branch from `main`.
2. Install dependencies with `npm install`.
3. Make your changes.
4. Run `npm run fmt` and `npm run lint` to ensure code quality.
5. Run `npm run check` to verify types.
6. Commit using [Conventional Commits](https://www.conventionalcommits.org/) (enforced by commitlint).
7. Open a Pull Request against `main`.

## Commit messages

This project uses [semantic-release](https://github.com/semantic-release/semantic-release), so commit messages must follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:

```
feat: add new icon weight support
fix: correct duotone opacity value
docs: update README examples
chore: bump @phosphor-icons/core
```

A commitlint hook runs automatically on every commit to validate the format.

## Regenerating icons

If you're updating the icon source (`@phosphor-icons/core`) or modifying the generation script:

```sh
npm run generate
```

This will rebuild every component in `src/icons/`. Include the regenerated files in your commit.

## Code style

- Formatting and linting are handled by [Biome](https://biomejs.dev/).
- Indent with **2 spaces**.
- Run `npm run fmt` before committing.
